## 一、项目概述
本项目为氢能组态上位机系统，基于**C/S架构（Winform）** 、**.NET 8.0（长期支持版）** 开发，采用**单项目+功能文件夹** 组织形式（无分层、无多类库），核心实现氢能生产线的全面监控、设备控制、配方管理、数据存储及权限管控。系统直连西门子S7-1200系列PLC，实时采集环境、工艺、设备状态数据，提供可视化操作、日志追溯、报表分析等能力，严格遵循氢能生产安全规范，满足产线安全、高效、可追溯的实际生产需求。

## 二、功能需求
### 2.1 核心模块功能
（1）**控制模块**

+ 总控制：支持生产线总启动、总停止、氢能压缩机复位、报警复位、空运行操作；
+ 单元控制：实现电解槽、纯化、储氢、加氢、冷却、检测、输送七大工位的独立启停；
+ 安全逻辑：严格遵循PLC底层互锁逻辑，保障总控制与单元控制的协同安全，防止误操作。

（2）**监控模块**

+ 环境监控：实时采集并展示厂房温度、湿度、氢气浓度，超标即时预警；
+ 设备状态：实时显示各工位核心设备（电解槽、纯化装置、储氢罐等）的运行/停止、过载、压力异常等状态；
+ 工艺参数：采集电解电流、纯化压力、储氢压力、加氢流量、氢气纯度等参数，精度保留小数点后1位；
+ 生产数据：动态统计并展示实时产氢量、合格氢量、生产能耗、累计报警次数。

（3）**配方模块**

+ 参数管理：支持电解电流范围、纯化压力上下限、储氢压力设定等工艺参数配置；
+ 场景绑定：配方与氢能应用场景（工业用氢、燃料电池用氢）关联，支持多场景差异化存储；
+ 操作功能：提供配方的添加、删除、修改、查询，及Excel格式的导入、导出、下发至PLC功能。

（4）**日志模块**

+ 日志记录：自动记录报警日志（时间、类型、工位）、操作日志（用户登录、控制操作、配方修改）；
+ 日志查询：支持按日期、日志类型筛选查询；
+ 日志导出：支持导出为TXT/Excel，支持直接打开日志存储目录。

（5）**报表模块**

+ 数据查询：按时间范围查询生产数据（产氢量、能耗）、工艺参数数据；
+ 报表生成：自动生成包含产氢量、氢气纯度、能耗、环境参数的综合报表；
+ 数据操作：支持Excel导出、报表分页查看，10万条以内数据导出≤3s。

（6）**图表模块**

+ 数据可视化：通过折线图展示电解电流、储氢压力、产氢量、能耗等数据的变化趋势；
+ 时间筛选：支持按开始/结束时间筛选图表数据，图表自适应界面显示。

（7）**参数模块**

+ PLC配置：支持IP、端口、CPU类型、机架号、插槽号、连接超时、重连间隔配置；
+ 软件配置：显示软件版本、设置试用时长；
+ 日志配置：支持定时清理日志（3/7/15/30/60天/不清理）、配置日志存储路径。

（8）**权限与用户模块**

+ 用户管理：添加、修改、删除用户，维护用户名、密码、角色信息；
+ 角色预设：管理员（全权限）、操作员（监控+部分控制）、访客（仅监控）；
+ 权限配置：支持修改各角色的模块操作权限（如控制/配方模块是否可操作）。

（9）**软件授权模块**

+ 授权类型：试用授权（可设置时长）、永久授权；
+ 授权验证：系统启动、总控制操作、配方修改时验证授权，未授权/过期限制功能使用。

### 2.2 非功能需求
（1）**界面需求**：大气简洁，关键信息（氢气浓度、产氢量、报警）突出显示；支持主题切换（蓝/绿/橙）、全局字体（微软雅黑/宋体）及缩放；适配1920×1080/1280×720分辨率，支持DPI自适应，无界面变形。  
（2）**性能需求**：PLC数据采集间隔可配置（默认300ms），采集成功率≥99.5%；界面操作响应≤500ms；连续运行72小时无卡顿/崩溃，PLC断连后自动重连（间隔可配置）。  
（3）**安全需求**：严格按角色权限控制功能访问，敏感操作（总停止、配方修改）验证身份；日志/配方数据本地存储，禁止未授权上传；控制操作提供二次提示，遵循PLC互锁逻辑防范氢气泄漏等风险。  
（4）**兼容性需求**：支持Windows 10操作系统；基于.NET 8.0 LTS开发；兼容西门子S7-1200系列PLC，以太网通讯。

## 三、项目整体设计（单项目+文件夹）
### 3.1 项目结构
创建**Windows窗体应用（.NET 8）** 项目，命名为`Tuling.HydrogenConfigurationSystem`，项目内按功能划分文件夹，所有代码、资源、配置均在该项目内，无额外类库/分层，结构简洁贴合工控上位机实际开发需求，具体文件夹及职责如下：

```plain
Tuling.HydrogenConfigurationSystem/
├─ Config/          # 配置文件目录（INI/JSON，存储PLC、系统、日志配置）
├─ Controls/        # 自定义控件目录（设备状态、参数显示、报警提示等用户控件）
├─ Data/            # 数据相关目录（数据库操作、实体类、本地数据存储）
│  ├─ Entities/     # 数据库实体类（用户、配方、日志、生产数据等）
│  └─ DB/           # 数据库操作类（MySQL直连，基于SqlSugarCore）
├─ Forms/           # 窗体目录（主窗体、各功能模块窗体、弹窗）
├─ PlcComm/# PLC通讯核心（西门子S7协议、数据读写、自动重连）
├─ Helpers/         # 通用工具类（Excel操作、日志记录、文件操作、授权验证等）
├─ Resources/       # 资源目录（图标、主题样式、默认配置模板等）
└─ Globals/         # 全局变量/常量（PLC客户端、全局字典、系统状态等）
```

### 3.2 核心设计原则
+ 无分层解耦：摒弃多层架构，采用“窗体直接调用功能类”的工控上位机经典开发模式，降低开发/维护复杂度；
+ 全局统一管理：通过Globals文件夹实现PLC客户端、实时数据字典、系统状态的全局统一访问；
+ 控件复用：自定义通用控件（设备状态、参数显示）放在Controls文件夹，减少重复开发；
+ 配置分离：PLC参数、系统参数通，数据库/日志配置通过JSON文件管理，便于现场调试修改；
+ 本地优先：所有数据（日志、配方、生产数据）均采用 MySQL 进行存储，无网络依赖，保障工业现场稳定性。

### 3.3 模块交互逻辑
+ **控制/监控模块：Forms窗体接收操作/展示请求，直接调用PlcComm中的类实现PLC数据读写，实时数据存入Globals全局字典，窗体从全局字典获取数据更新界面；【可以用作面试设计亮点】**
+ 配方/用户/权限模块：Forms窗体发起操作请求，直接调用Data/DB中的数据库操作类实现数据增删改查，配合Helpers中的工具类实现Excel导入导出；
+ 日志/报表/图表模块：日志由Helpers/NLog自动记录，报表/图表通过Data/DB查询历史数据 Scottplot/Scottplot【**可以用作面试设计亮点**】，结合Helpers中的Excel工具/图表控件实现数据可视化与导出；**【可以用作面试设计亮点，数据库优化：分页查询、索引优化】**
+ 授权/参数模块：授权验证由Helpers中的授权类实现（加密算法），参数配置由Forms窗体修改后，直接写入Config文件夹的配置文件，系统启动时加载。

## 四、核心模块详细设计
### 4.1 PLC通讯模块（PlcComm/）
#### 4.1.1 通讯协议与核心功能
+ 通讯协议：西门子S7协议（基于HslCommunication组件），支持以太网直连西门子S7-1200；
+ 核心功能：**批量读取（通过字典）**PLC设备状态、工艺/环境参数，**批量写入（通过字典）**控制指令、配方参数；实现PLC断连自动重连（间隔可配置）；
+ 数据映射：通过Globals中的 2  个全局字典实现PLC地址与业务数据的映射，避免硬编码：
    - `WriteDic`：Key=参数名称（如电解电流上限），Value=PLC地址，存储需要写入的PLC地址；
    - `ReadDic`：Key=参数名称（如氢气浓度、储氢压力），Value=实时值，存储PLC读取的实时数据，供所有窗体调用。

#### 4.1.2 核心类设计
+ `PlcClientHelper.cs`：PLC通讯核心类，封装PLC连接、断开、批量读、批量写、自动重连逻辑，提供静态方法供全局调用；
+ `PlcVarConfig.cs`：PLC变量配置模型，对应Config文件夹中的PLC变量表Excel，存储参数名称、PLC地址、数据类型。

### 4.2 数据与数据库模块（Data/）
#### 4.2.1 数据库选型
**MySQL 8.0  数据库**，选用 SqlSugarCore ORM 框架。
- Admin / 123
- Operator / 123
- Visitor / 123

#### 4.2.2 核心实体类（Data/Entities/）
所有实体类与数据库表一一对应，基于SqlSugarCore实现ORM映射，字段名统一使用英文，贴合C#开发规范，核心实体如下：

1. `UserEntity.cs`（用户表）：Id、UserName、UserPassword、Role（角色：Admin/Operator/Visitor）
2. `RoleAuthEntity.cs`（角色权限表）：Id、Role、ControlAuth、MonitorAuth、RecipeAuth等（各模块权限布尔值）
3. `RecipeEntity.cs`（配方表）：Id、ApplicationScene（应用场景）、ElectrolysisCurrentMin/Max（电解电流范围）、PurificationPressureMin/Max（纯化压力上下限）等工艺参数
4. `LogAlarmEntity.cs`（报警日志表）：Id、AlarmTime、AlarmType、AlarmStation（报警工位）、AlarmContent（报警内容）
5. `LogOperateEntity.cs`（操作日志表）：Id、OperateTime、UserName、OperateModule、OperateContent
6. `ProductionDataEntity.cs`（生产数据表）：Id、InsertTime、ElectrolysisCurrent（电解电流）、HydrogenOutput（产氢量）、EnergyConsumption（能耗）等全量监控数据

#### 4.2.3 数据库操作类（Data/DB/）
每个实体类对应一个操作类，封装SqlSugarCore的CRUD操作，提供同步/异步方法，供窗体直接调用，核心类如下：

+ `DbContext.cs`：数据库上下文，初始化SqlSugarClient，配置连接字符串、自动建表，提供全局数据库实例，通过配置文件来进行读取的；
+ `UserService.cs`、`RecipeService.cs`、`LogService.cs`、`ProductionDataService.cs`：各表对应的操作类，封装增删改查、条件查询、批量导入导出等业务方法。

### 4.3 权限与授权模块
#### 4.3.1 权限控制
+ 角色预设：管理员（Admin）、操作员（Operator）、访客（Visitor），角色信息存储在`UserEntity`，权限配置存储在`RoleAuthEntity`；
+ 界面控制：窗体加载时，根据当前登录用户的角色，从`RoleAuthEntity`获取权限，动态显示/隐藏操作控件（如访客隐藏控制按钮、操作员隐藏配方修改按钮）；
+ 操作验证：敏感操作（总停止、配方修改、用户管理）执行前，再次验证用户权限，无权限则弹窗提示并禁止执行。

#### 4.3.2 授权验证（Helpers/AuthHelper.cs）
+ 授权存储：授权信息加密存储在Config文件夹的`auth.json`，包含授权类型（试用/永久）、试用截止时间、设备标识；
+ 验证时机：系统启动时、执行总控制操作时、修改配方/权限时，自动调用授权验证方法；
+ 权限限制：未授权/授权过期时，限制控制模块、配方模块、用户管理模块的使用，仅开放监控功能。

### 4.4 UI界面设计（Forms/ + Controls/）
#### 4.4.1 整体布局
基于 SunnyUI 控件库实现，主窗体继承`UIHeaderAsideMainFooterFrame`，分为四部分，适配工控机操作习惯：

+ 顶栏：展示系统名称、当前用户、核心实时数据（氢气浓度、产氢量、系统状态）、报警提示（红色闪烁显示）；
+ 侧边栏：功能模块导航（控制、监控、配方、日志等），基于`UINavBar`实现，点击切换中间栏窗体；
+ 中间栏：功能窗体展示区，加载对应模块的窗体（如`FrmDeviceControl`、`FrmRealTimeMonitor`）；
+ 底栏：展示PLC连接状态（指示灯）、CPU/内存占用、软件版本、授权状态、系统时间。

#### 4.4.2 核心窗体（Forms/）
+ `FrmMain.cs`：主窗体，初始化全局配置、PLC连接、侧边栏导航、权限加载；
+ `FrmDeviceControl.cs`：设备控制窗体，实现总控制、各工位单元控制，按钮绑定PLC写入逻辑；
+ `FrmRealTimeMonitor.cs`：实时监控窗体，按工位分组展示设备状态、工艺参数，调用自定义控件实现可视化；
+ `FrmRecipeManager.cs`：配方管理窗体，实现配方的增删改查及Excel导入导出、下发PLC；
+ `FrmLogQuery.cs`：日志查询窗体，支持多条件筛选、日志导出、打开存储目录；
+ `FrmReportChart.cs`：报表图表窗体，实现数据查询、报表生成、折线图可视化、Excel导出；
+ `FrmSystemConfig.cs`：系统配置窗体，实现PLC参数、软件参数、日志配置的修改与保存；
+ `FrmUserManager.cs`：用户权限窗体，实现用户增删改查、角色权限配置。

#### 4.4.3 自定义控件（Controls/）
封装通用可视化控件，添加自定义属性，支持设计时配置，直接拖放使用，核心控件：

+ `UC_DeviceState.cs`：设备状态控件，包含状态名称、运行指示灯（绿=运行/红=停止/黄=报警），自定义属性`DeviceName`、`DeviceState`；
+ `UC_ParamShow.cs`：参数显示控件，包含参数名称、实时值、单位、阈值预警（超限时字体变红），自定义属性`ParamName`、`ParamValue`、`MaxValue`、`MinValue`；
+ `UC_AlarmTip.cs`：报警提示控件，实时显示最新报警信息，支持红色闪烁、报警声音提示。

### 4.5 通用工具模块（Helpers/）
封装项目通用工具方法，提供静态方法供全局调用，核心类：

+ `ExcelHelper.cs`：基于MiniExcel实现Excel导入导出，支持配方、报表、日志的Excel操作，无需安装Office；
+ `LogHelper.cs`：基于NLog 库实现日志记录，封装信息、警告、错误日志的记录方法，配置日志存储路径、分割规则；
+ `FileHelper.cs`：文件操作工具，实现配置文件读写、日志文件清理、文件夹打开等；
+ `AuthHelper.cs`：授权验证工具，实现授权信息加密、解密、验证、试用时长设置；
+ `UIHelper.cs`：UI辅助工具，实现主题切换、字体设置、弹窗提示、跨线程更新窗体控件（解决PLC线程更新UI跨线程问题）。

### 4.6 配置文件模块（Config/）
所有配置文件均存储在此，支持现场直接修改，无需重新编译项目，核心配置：

+ `plc_config.ini`：PLC连接参数（IP、端口、CPU类型、采集间隔、重连间隔等）；
+ `system_config.ini`：软件配置（试用时长、日志清理天数、日志存储路径、默认字体等）；
+ `auth.json`：授权信息（加密存储，含授权类型、截止时间、设备标识）；
+ `nlog.config`：NLog日志配置（存储格式、分割规则、保存路径等）；
+ `plc_var_config.xlsx`：PLC变量表（参数名称、PLC地址、数据类型，系统启动时加载到全局字典）。

### 4.7 全局模块（Globals/）
存储项目全局变量、常量、枚举，供所有模块直接调用，核心类：

+ `GlobalConstants.cs`：全局常量（如氢气爆炸下限4%、默认采集间隔300ms、默认DPI缩放100%）；
+ `GlobalVariables.cs`：全局变量（PLC客户端实例、2 个核心字典ReadDic/WriteDic、当前登录用户、PLC连接状态、系统授权状态）；
+ `GloablEnums.cs`：全局枚举（如设备状态、报警类型、角色类型、授权类型、PLC数据类型）。

## 六、核心技术实现（关键代码片段）
### 6.1 PLC连接与数据读取（PlcComm/PlcClientHelper.cs + Globals/GlobalVariables.cs）
```csharp
// Globals/GlobalVariables.cs 全局PLC相关变量
using HslCommunication.Profinet.Siemens;
using System.Collections.Generic;

namespace Tuling.HydrogenConfigurationSystem.Globals
{
    public static class GlobalVariables
    {
        // PLC连接参数（从plc_config.ini加载）
        public static string PlcIp { get; set; } = "127.0.0.1";
        public static int PlcPort { get; set; } = 102;
        public static SiemensVersion CpuType { get; set; } = SiemensVersion.S71200;
        public static byte Rack { get; set; } = 0;
        public static byte Slot { get; set; } = 1;
        public static int ReadInterval { get; set; } = 300; // 默认采集间隔300ms

using HslCommunication;
using Tuling.HydrogenConfigurationSystem.Globals;

namespace Tuling.HydrogenConfigurationSystem.PlcComm
{
    public static class PlcClientHelper
    {
        /// <summary>
        /// 初始化PLC连接
        /// </summary>
        /// <returns>连接结果</returns>
        public static OperateResult InitPlcConnection()
        {
            try
            {
                GlobalVariables.PlcClient.SetConnectionParam(GlobalVariables.PlcIp, GlobalVariables.PlcPort);
                GlobalVariables.PlcClient.SiemensVersion = GlobalVariables.CpuType;
                GlobalVariables.PlcClient.Rack = GlobalVariables.Rack;
                GlobalVariables.PlcClient.Slot = GlobalVariables.Slot;
                // 建立连接
                var result = GlobalVariables.PlcClient.ConnectServer();
                GlobalVariables.PlcIsConnected = result.IsSuccess;
                return result;
            }
            catch (Exception ex)
            {
                return new OperateResult(ex.Message);
            }
        }

        /// <summary>
        /// 批量读取PLC数据
        /// </summary>
        /// <returns>读取结果</returns>
        public static OperateResult<Dictionary<string, object>> BatchReadPlcData()
        {
            if (!GlobalVariables.PlcIsConnected) return new OperateResult<Dictionary<string, object>>("PLC未连接");
            try
            {
                // 批量读取全局字典中的PLC地址
                var result = GlobalVariables.PlcClient.BatchRead(GlobalVariables.ReadDic);
                if (result.IsSuccess)
                {
                    // 将读取结果更新到全局DataDic
                    foreach (var item in result.Content)
                    {
                        var paramName = GlobalVariables.ReadDic.First(kv => kv.Key == item.Key).Key;
                        if (GlobalVariables.DataDic.ContainsKey(paramName))
                            GlobalVariables.DataDic[paramName] = item.Value;
                        else
                            GlobalVariables.DataDic.Add(paramName, item.Value);
                    }
                }
                return result;
            }
            catch (Exception ex)
            {
                return new OperateResult<Dictionary<string, object>>(ex.Message);
            }
        }

        /// <summary>
        /// 批量写入PLC数据
        /// </summary>
        /// <param name="writeData">写入数据（参数名称-值）</param>
        /// <returns>写入结果</returns>
        public static OperateResult BatchWritePlcData(Dictionary<string, object> writeData)
        {
            if (!GlobalVariables.PlcIsConnected) return new OperateResult("PLC未连接");
            try
            {
                // 转换为PLC地址-值的字典
                var plcWriteData = new Dictionary<string, object>();
                foreach (var item in writeData)
                {
                    if (GlobalVariables.WriteDic.ContainsKey(item.Key))
                        plcWriteData.Add(GlobalVariables.WriteDic[item.Key], item.Value);
                }
                // 批量写入
                var result = GlobalVariables.PlcClient.BatchWrite(plcWriteData);
                return result;
            }
            catch (Exception ex)
            {
                return new OperateResult(ex.Message);
            }
        }

        /// <summary>
        /// 断开PLC连接
        /// </summary>
        public static void DisconnectPlc()
        {
            if (GlobalVariables.PlcIsConnected)
            {
                GlobalVariables.PlcClient.ConnectClose();
                GlobalVariables.PlcIsConnected = false;
            }
        }

        /// <summary>
        /// PLC自动重连
        /// </summary>
        public static async Task AutoReConnect()
        {
            while (!GlobalVariables.Cts.Token.IsCancellationRequested)
            {
                if (!GlobalVariables.PlcIsConnected)
                {
                    var result = InitPlcConnection();
                    if (result.IsSuccess)
                    {
                        Helpers.UIHelper.ShowTips("PLC重连成功");
                    }
                }
                await Task.Delay(GlobalVariables.ReConnectInterval);
            }
        }
    }
}
```

## 八、关键测试与问题排查
### 8.1 核心测试要点
1. PLC通讯测试：验证PLC连接、批量读写、自动重连、数据采集成功率是否≥99.5%；
2. 安全功能测试：验证氢气浓度超标报警、PLC互锁逻辑、敏感操作权限验证是否有效；
3. 性能测试：验证连续运行72小时无卡顿/崩溃，报表生成/Excel导出（10万条数据）≤3s；
4. 界面适配测试：验证在1920×1080/1280×720分辨率下界面无变形，DPI缩放自适应；
5. 授权测试：验证未授权/授权过期时，控制/配方模块是否被限制，仅开放监控功能。

### 8.2 常见问题排查
1. PLC连接失败：检查PLC IP/端口配置是否正确，工控机与PLC是否在同一网段，防火墙是否放行102端口；
2. 数据采集异常：检查`plc_var_config.xlsx`中PLC地址/数据类型是否正确，全局字典是否正常加载；
3. 界面跨线程报错：所有PLC线程更新UI的操作，必须通过`UIHelper.InvokeControl`实现；
4. 数据库操作失败：检查工控机对软件目录的读写权限，确保`HydrogenData.db`可正常创建/修改；
5. 界面变形：将窗体`AutoScaleMode`设置为`None`，在`app.manifest`中配置DPI感知为`true`；
6. 报警不触发：检查氢气浓度阈值设置是否正确（4%），PLC数据读取是否正常，报警逻辑代码是否执行。

## 九、电气地址表
本地址表基于**西门子S7-1200** PLC编制，从**DB1.1**（字节1）开始连续分配地址，适配氢能生产线七大工位（电解槽、纯化、储氢、加氢、冷却、检测、输送）的**控制指令、设备状态、工艺参数、环境参数、生产数据、报警信号**六大类数据，完全匹配上位机的PLC通讯、监控、控制功能需求。  
地址格式遵循西门子标准：`DB1.DBXx.y`（位地址/BOOL）、`DB1.DBDx`（双字地址/REAL/DINT）、`DB1.DBWx`（字地址/INT），**读写属性**以上位机视角标注（W=上位机写/PLC读，R=上位机读/PLC写，RW=双向读写），工艺参数精度均按上位机要求配置为**小数点后1位**，氢气浓度等安全参数标注量程及阈值。

:::info
I、Q、M则代表不同类型的存储区

I：输入存储区，接收外部输入信号，比如传感器

Q: 输出存储区，输出信号到外部负载，如继电器、指示灯

M: 中间存储区，用于存储程序执行过程中的中间状态

DB1.DBX1.7 代表的是数据块 DB1 中第1个字节的第7位。

这里的DB1指的是数据块的编号为1，DBX表示数据块中的数据位，1.7表示第1个字节的第7位。这种表示方法用于访问和操作数据块中的具体位。

### 9.3 PLC端配套建议
1. **DB1块设置**：在TIA Portal中创建DB1，**取消勾选优化的块访问**，块类型为「全局数据块」，存储区选择「保持」（掉电保持生产数据、配方参数）；
2. **互锁逻辑**：在PLC梯形图/FBD中实现**总停止优先、总启动前置、配方参数限幅**逻辑，避免上位机误操作；
3. **数据转换**：PLC端完成**传感器原始值→工程量**的转换（如4-20mA→0~10%VOL氢气浓度），上位机直接读取显示；
4. **掉电保持**：将累计产氢量、累计能耗、报警次数等关键生产数据设置为**PLC掉电保持**；
5. **报警触发**：PLC端实现报警信号的**边沿检测+自锁**，上位机复位后解除自锁；
6. **通讯优化**：PLC端开启**以太网心跳包**，与上位机保持通讯，超时则触发通讯中断报警。

```csharp
using System;
namespace Tuling.HydrogenConfigurationSystem.Globals
{
    /// <summary>
    /// PLC变量配置模型（与Config/plc_var_config.xlsx列名严格一致）
    /// </summary>

    {
        /// <summary>
        /// 变量名称（如：厂房氢气浓度）
        /// </summary>

        /// <summary>
        /// PLC地址（如：DB1.DBD19、DB1.DBX1.0）
        /// </summary>
        public string PLC地址 { get; set; } = string.Empty;

        /// <summary>
        /// 数据类型（Bool/Real/Int/Dint，与HslCommunication匹配）
        /// </summary>
        public string 数据类型 { get; set; } = string.Empty;

        /// <summary>
        /// 读写属性（W=上位机写/R=上位机读/RW=双向）
        /// </summary>
        public string 读写属性 { get; set; } = string.Empty;

        /// <summary>
        /// 所属模块（如：控制模块、环境参数模块）
        /// </summary>
        public string 所属模块 { get; set; } = string.Empty;

        /// <summary>
        /// 采集间隔(ms)，-表示无采集（控制指令）
        /// </summary>
        public string 采集间隔 { get; set; } = "-";

        /// <summary>
        /// 备注（阈值、掉电保持等）
        /// </summary>
        public string 备注 { get; set; } = string.Empty;

        /// <summary>
        /// 转换为HslCommunication数据类型枚举（内部使用）
        /// </summary>
        public HslCommunication.Core.Types.DataTypeEnum GetHslDataType()
        {
            return 数据类型.ToUpper() switch
            {
                "BOOL" => HslCommunication.Core.Types.DataTypeEnum.Bit,
                "REAL" => HslCommunication.Core.Types.DataTypeEnum.Float,
                "INT" => HslCommunication.Core.Types.DataTypeEnum.Int16,
                "DINT" => HslCommunication.Core.Types.DataTypeEnum.Int32,
                _ => HslCommunication.Core.Types.DataTypeEnum.Float
            };
        }

        /// <summary>
        /// 获取采集间隔（转int，-则返回0）
        /// </summary>
        public int GetCollectInterval()
        {
            if (int.TryParse(采集间隔, out int interval) && interval > 0)
                return interval;
            return 0;
        }
    }
```

<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">在原有</font>`<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">PlcClientHelper.cs</font>`<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">中添加</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">LoadPlcVarConfig</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">方法（自动读取 Excel、初始化 3 个全局字典），并补充原有方法的调用逻辑，</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">完整代码如下</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">（直接覆盖原有文件）：</font>

```csharp
using HslCommunication;
using HslCommunication.Core.Types;
using MiniExcelLibs;
using System.IO;
using Tuling.HydrogenConfigurationSystem.Globals;
using Tuling.HydrogenConfigurationSystem.Helpers;

namespace Tuling.HydrogenConfigurationSystem.PlcComm
{
    /// <summary>
    /// PLC通讯核心助手类（含Excel配置表加载、批量读写、自动重连）
    /// </summary>
    public static class PlcClientHelper
    {
        #region 核心新增：加载PLC变量配置Excel并初始化全局字典
        /// <summary>
        /// 加载Config/plc_var_config.xlsx，初始化ReadDic/WriteDic/DataDic
        /// </summary>
        /// <returns>加载结果（成功/失败信息）</returns>
        public static OperateResult LoadPlcVarConfig()
        {
            try
            {
                // 1. 检查Excel文件是否存在
                if (!File.Exists(GlobalVariables.PlcVarConfigPath))
                {
                    string errorMsg = $"PLC变量配置文件不存在：{GlobalVariables.PlcVarConfigPath}";
                    LogHelper.Error(errorMsg);
                    UIHelper.ShowError(errorMsg);
                    return new OperateResult(errorMsg);
                }

                // 2. 读取Excel为PlcVarConfigModel列表
                GlobalVariables.PlcVarConfigList = MiniExcel.Query<PlcVarConfigModel>(GlobalVariables.PlcVarConfigPath).ToList();
                if (GlobalVariables.PlcVarConfigList.Count == 0)
                {
                    string errorMsg = "PLC变量配置Excel无有效数据，请检查文件内容";
                    LogHelper.Error(errorMsg);
                    UIHelper.ShowError(errorMsg);
                    return new OperateResult(errorMsg);
                }

                // 3. 清空原有全局字典，重新初始化
                GlobalVariables.ReadDic.Clear();
                GlobalVariables.WriteDic.Clear();
                GlobalVariables.DataDic.Clear();

                // 4. 遍历配置列表，填充3个全局字典
                foreach (var varModel in GlobalVariables.PlcVarConfigList)
                {
                    // 过滤空地址/空名称
                    if (string.IsNullOrWhiteSpace(varModel.PLC地址) || string.IsNullOrWhiteSpace(varModel.变量名称))
                        continue;

                    // 初始化DataDic（所有变量默认值NA，界面展示用）
                    if (!GlobalVariables.DataDic.ContainsKey(varModel.变量名称))
                        GlobalVariables.DataDic.Add(varModel.变量名称, "NA");

                    // 初始化WriteDic（W/RW属性：变量名称→PLC地址，供上位机下发指令/配方）
                    if (varModel.读写属性.ToUpper() is "W" or "RW")
                    {
                        if (!GlobalVariables.WriteDic.ContainsKey(varModel.变量名称))
                            GlobalVariables.WriteDic.Add(varModel.变量名称, varModel.PLC地址);
                    }

                    // 初始化ReadDic（R/RW属性：PLC地址→Hsl数据类型，供批量读取PLC数据）
                    if (varModel.读写属性.ToUpper() is "R" or "RW")
                    {
                        if (!GlobalVariables.ReadDic.ContainsKey(varModel.PLC地址))
                            GlobalVariables.ReadDic.Add(varModel.PLC地址, varModel.GetHslDataType());
                    }
                }

                // 5. 加载成功，记录日志+提示
                string successMsg = $"PLC变量配置表加载成功，共加载{GlobalVariables.PlcVarConfigList.Count}个变量，可读取{GlobalVariables.ReadDic.Count}个地址，可写入{GlobalVariables.WriteDic.Count}个地址";
                LogHelper.Info(successMsg);
                UIHelper.ShowTips(successMsg);
                return OperateResult.Success;
            }
            catch (Exception ex)
            {
                string errorMsg = $"加载PLC变量配置表失败：{ex.Message}";
                LogHelper.Error(errorMsg, ex);
                UIHelper.ShowError(errorMsg);
                return new OperateResult(errorMsg);
            }
        }
        #endregion

        #region 原有方法（保留+少量优化）
        /// <summary>
        /// 初始化PLC连接（基于全局配置的IP/端口/CPU类型）
        /// </summary>
        /// <returns>连接结果</returns>
        public static OperateResult InitPlcConnection()
        {
            try
            {
                GlobalVariables.PlcClient?.ConnectClose();
                GlobalVariables.PlcClient = new SiemensS7Client();
                GlobalVariables.PlcClient.SetConnectionParam(GlobalVariables.PlcIp, GlobalVariables.PlcPort);
                GlobalVariables.PlcClient.SiemensVersion = GlobalVariables.CpuType;
                GlobalVariables.PlcClient.Rack = GlobalVariables.Rack;
                GlobalVariables.PlcClient.Slot = GlobalVariables.Slot;

                // 建立连接
                var result = GlobalVariables.PlcClient.ConnectServer();
                GlobalVariables.PlcIsConnected = result.IsSuccess;

                if (result.IsSuccess)
                {
                    LogHelper.Info($"PLC连接成功：{GlobalVariables.PlcIp}:{GlobalVariables.PlcPort}");
                }
                else
                {
                    LogHelper.Error($"PLC连接失败：{result.Message}，地址：{GlobalVariables.PlcIp}:{GlobalVariables.PlcPort}");
                }
                return result;
            }
            catch (Exception ex)
            {
                string errorMsg = $"PLC连接初始化异常：{ex.Message}";
                LogHelper.Error(errorMsg, ex);
                return new OperateResult(errorMsg);
            }
        }

        /// <summary>
        /// 批量读取PLC数据（自动更新全局DataDic）
        /// </summary>
        /// <returns>读取结果（含数据字典）</returns>
        public static OperateResult<Dictionary<string, object>> BatchReadPlcData()
        {
            if (!GlobalVariables.PlcIsConnected)
            {
                string errorMsg = "PLC未连接，无法读取数据";
                return new OperateResult<Dictionary<string, object>>(errorMsg);
            }

            if (GlobalVariables.ReadDic.Count == 0)
            {
                string errorMsg = "PLC读取字典为空，请先加载变量配置表";
                LogHelper.Warn(errorMsg);
                return new OperateResult<Dictionary<string, object>>(errorMsg);
            }

            try
            {
                var readResult = GlobalVariables.PlcClient.BatchRead(GlobalVariables.ReadDic);
                if (readResult.IsSuccess)
                {
                    // 遍历读取结果，更新全局DataDic（PLC地址→变量名称→值）
                    foreach (var plcVar in GlobalVariables.PlcVarConfigList)
                    {
                        if (readResult.Content.ContainsKey(plcVar.PLC地址) &&
                            GlobalVariables.DataDic.ContainsKey(plcVar.变量名称))
                        {
                            GlobalVariables.DataDic[plcVar.变量名称] = readResult.Content[plcVar.PLC地址];
                        }
                    }
                }
                else
                {
                    LogHelper.Warn($"PLC批量读取失败：{readResult.Message}");
                    GlobalVariables.PlcIsConnected = false;
                }
                return readResult;
            }
            catch (Exception ex)
            {
                string errorMsg = $"PLC批量读取异常：{ex.Message}";
                LogHelper.Error(errorMsg, ex);
                GlobalVariables.PlcIsConnected = false;
                return new OperateResult<Dictionary<string, object>>(errorMsg);
            }
        }

        /// <summary>
        /// 批量写入PLC数据（基于变量名称，自动匹配PLC地址）
        /// </summary>
        /// <param name="writeData">变量名称-值 字典（如：{"生产线总启动", true}）</param>
        /// <returns>写入结果</returns>
        public static OperateResult BatchWritePlcData(Dictionary<string, object> writeData)
        {
            if (!GlobalVariables.PlcIsConnected)
                return new OperateResult("PLC未连接，无法写入数据");

            if (GlobalVariables.WriteDic.Count == 0)
                return new OperateResult("PLC写入字典为空，请先加载变量配置表");

            if (writeData == null || writeData.Count == 0)
                return new OperateResult("写入数据为空");

            try
            {
                // 转换为：PLC地址-值 字典（适配Hsl批量写入）
                var plcWriteDic = new Dictionary<string, object>();
                foreach (var (varName, varValue) in writeData)
                {
                    if (GlobalVariables.WriteDic.ContainsKey(varName))
                    {
                        plcWriteDic.Add(GlobalVariables.WriteDic[varName], varValue);
                        // 记录写入日志
                        LogHelper.Info($"PLC写入指令：{varName} → {varValue}，地址：{GlobalVariables.WriteDic[varName]}");
                    }
                }

                if (plcWriteDic.Count == 0)
                    return new OperateResult("无有效写入地址，请检查变量名称");

                var writeResult = GlobalVariables.PlcClient.BatchWrite(plcWriteDic);
                if (!writeResult.IsSuccess)
                {
                    LogHelper.Error($"PLC批量写入失败：{writeResult.Message}");
                }
                return writeResult;
            }
            catch (Exception ex)
            {
                string errorMsg = $"PLC批量写入异常：{ex.Message}";
                LogHelper.Error(errorMsg, ex);
                return new OperateResult(errorMsg);
            }
        }

        /// <summary>
        /// 断开PLC连接
        /// </summary>
        public static void DisconnectPlc()
        {
            if (GlobalVariables.PlcIsConnected && GlobalVariables.PlcClient != null)
            {
                GlobalVariables.PlcClient.ConnectClose();
                GlobalVariables.PlcIsConnected = false;
                LogHelper.Info("PLC连接已主动断开");
            }
        }

        /// <summary>
        /// PLC自动重连（后台异步运行）
        /// </summary>
        public static async Task AutoReConnect()
        {
            while (!GlobalVariables.Cts.Token.IsCancellationRequested)
            {
                if (!GlobalVariables.PlcIsConnected)
                {
                    LogHelper.Warn("PLC断开连接，开始自动重连...");
                    var reConnResult = InitPlcConnection();
                    if (reConnResult.IsSuccess)
                    {
                        UIHelper.ShowTips("PLC自动重连成功");
                    }
                }
                await Task.Delay(GlobalVariables.ReConnectInterval);
            }
        }
        #endregion
    }
}
```

可能的问题  
提示PLC 变量配置文件不存在

1. 检查 Config 文件夹是否有 plc_var_config.xlsx；

2. 检查文件属性是否设置为「如果较新则复制」；

3. 查看输出目录（bin/Debug/net8.0-windows/Config）是否有该 Excel  


加载成功但ReadDic/WriteDic 为 0	

1. 检查 Excel 中读写属性列是否为 R/W/RW；

2. 检查PLC 地址列是否为有效格式（如 DB1.DBX1.0、DB1.DBD19）；

3. 检查 Excel 无空行 / 空列

  
批量读写提示无有效地址	

1. 检查变量名称是否与 Excel 中变量名称列完全一致（含空格 / 符号）；

2. 检查该变量的读写属性是否匹配（写操作需 W/RW，读操作需 R/RW）  


数据显示NA/---	

1. 检查 PLC 是否正常连接（ledPlcState 为绿色）；

2. 检查 PLC 端是否已配置对应地址的数据；

3. 检查 Excel 中数据类型是否与 PLC 端一致
